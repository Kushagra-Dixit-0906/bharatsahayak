[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let me walk you through our system architecture. 
When a query enters, it first goes through the Security Checkpoint node, which scrubs sensitive PII like Aadhaar cards and filters out malicious prompt injections. 
Next, the query reaches the load_farmer_profile node, which extracts user details and updates the farmer's state in memory. 
The orchestrator agent then takes over, analyzing the query and delegating it to one of our four specialized agents: the farming_advisor, weather_advisor, gov_schemes_advisor, or crop_disease_advisor. 
These specialized agents are backed by a custom MCP Server that provides live data. 
Before outputting, the query goes through the hitl_checkpoint. If key details like the farmer's state or land acreage are missing, the system uses RequestInput to pause and ask the farmer to clarify. Once resumed, the formatted final output node streams the clean response back to the user.

Let's look at this in action. 
I will type: "I am a wheat farmer in Punjab. Tell me what I should do right now for my wheat crop." 
The load_farmer_profile node immediately captures the location as Punjab and crop as wheat. The orchestrator delegates to the weather advisor, which connects to our MCP Server to fetch Punjab's live weather. It advises me to apply urea and perform light watering because of the sunny 32-degree forecast.

Now let's try diagnosing a disease. 
I'll input: "My rice crop has some brown spots on the leaves. What disease could this be and how do I treat it?" 
The orchestrator automatically routes this to the crop disease advisor, which queries our crop disease database on the MCP Server. It correctly identifies Brown Spot Disease and lists treatments like Mancozeb and preventative steps.

Lastly, let's verify our security checkpoint. 
If a user writes: "My Aadhaar card is 1234 5678 9012 and my bank pin is 9999. Can you tell me what government schemes I qualify for?" 
The security checkpoint scrubs the Aadhaar ID and flags the phrase "bank pin" as a critical risk. The orchestrator is bypassed entirely, and the system blocks the request, outputting: "Security Alert: Your query contains sensitive financial requests or off-topic unsafe instructions. Action blocked."
